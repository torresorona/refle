# Development guide

## Prerequisites

- **Docker** + Docker Compose (for Postgres, Redis, MinIO)
- **Node 20+** and npm
- **[uv](https://docs.astral.sh/uv/)** for the Python workspace
  (`curl -LsSf https://astral.sh/uv/install.sh | sh`)

## First-time setup

```bash
cp .env.example .env
make install        # npm install + uv sync --all-packages
make up             # start infra (Postgres+pgvector, Redis, MinIO)
make migrate        # apply database migrations
```

## Running the apps

In separate terminals:

```bash
make api            # FastAPI -> http://localhost:8000  (docs at /docs)
make web            # Next.js -> http://localhost:3000
make worker         # Celery worker (optional until Phase 2)
```

Verify the stack:

```bash
curl -s localhost:8000/health        # {"status":"ok"}
curl -s localhost:8000/health/db     # {"status":"ok","database":"ok"}
curl -s localhost:8000/meta          # version, license tier, AI model, registries
```

## Database migrations

Migrations live in `services/api/migrations`. The async Alembic env reads the database
URL and model metadata from refle settings.

```bash
make migrate                          # alembic upgrade head
make revision msg="add evidence"      # autogenerate a new migration
```

The baseline migration `0001_pgvector` enables the `vector` extension; the domain schema
is autogenerated on top of it.

## AI provider configuration

The gateway in `libs/ai_core` is provider-agnostic. Configure via `.env`:

```bash
REFLE_AI_PROVIDER=gemini             # gemini | openai | local
REFLE_AI_MODEL=gemini-3.5-flash
GEMINI_API_KEY=...                   # required for the gemini provider
```

For **sovereign / local** mode, point at any OpenAI-compatible endpoint (e.g. Ollama):

```bash
REFLE_AI_PROVIDER=local
REFLE_AI_LOCAL_BASE_URL=http://localhost:11434/v1
REFLE_AI_LOCAL_MODEL=llama3.1
```
